# Чат с шифрованием — руководство пользователя

Консольный мессенджер на FASM (x86-64, Linux) с RSA шифрованием.

> Работает только на Linux (или WSL на Windows).

---

## Полезные команды

```bash
# Сборка
make

# Удалить бинарники
make clean

# Перезапустить сервер (убивает старый процесс, освобождает порт, запускает новый)
make restart        # или: bash restart.sh

# Удалить все ключи (.key) и базу данных (data.bin)
make cleandata

# Запустить автотест (6 параллельных клиентов, все фазы)
bash run_test.sh

# Запустить симуляцию живого диалога (4 пользователя, 12 сообщений)
bash run_test.sh test2.py
```

---

## Шаг 1 — Запуск сервера

```bash
./build/server
```

Сервер запускается на порту **8080**. Держи его в отдельном терминале.
При перезапуске данные восстанавливаются из `data.bin`.

---

## Шаг 2 — Запуск клиента

```bash
./build/client
```

Клиент подключается к `127.0.0.1:8080`.

---

## Шаг 3 — Вход или регистрация

```
1. Войти  2. Регистрация:
```

- **Первый раз** — выбери `2`. Вводишь имя пользователя, клиент генерирует RSA-ключи, регистрирует тебя на сервере и сохраняет `username.key` локально.
- **Повторный вход** — выбери `1`. Вводишь имя — клиент загружает `username.key` и проходит авторизацию.

> Пустое имя пользователя — клиент завершается.

---

## Главное меню

```
=== МЕНЮ ===
1. Создать чат
2. Войти в чат
3. Пригласить в чат
4. Выйти
```

**1 — Создать чат** — вводишь название (Enter без текста = отмена). После создания показывается ID нового чата, затем возврат в меню.

**2 — Войти в чат** — показывает только твои чаты (в которые тебя пригласили). Вводишь номер из списка или Enter для возврата в меню.

**3 — Пригласить в чат** — показывает твои чаты, вводишь номер чата и имя пользователя. Пользователь должен быть зарегистрирован, а ты должен быть членом чата.

**4 — Выйти** — закрывает соединение.

---

## Внутри чата

После входа в чат автоматически загружаются все сообщения в формате `имя: текст`. Сверху заголовок с именем чата:

```
=== ЧАТ: mychat
alice: Привет!
bob: Привет, Alice!
Введите Сообщение (Enter = выйти):
```

- **Текст** — отправить сообщение (до 252 символов), чат обновится
- **Enter (пусто)** — выйти в главное меню

## Ограничения

| Параметр | Значение |
|---|---|

| Максимум чатов | 16 |
| Максимум ключей чатов | 128 |

| Длина имени пользователя | до 31 символа (отображается полностью) |

## Makefile

| Цель | Что делает |
|---|---|
| `make` | Собирает `server`, `client`, `keygen` через FASM + ld в папку `build/` |
| `make clean` | Удаляет бинарники и `.o` файлы из `build/` |
| `make cleandata` | Удаляет `build/data.bin` и все `build/*.key` (база и ключи пользователей) |
| `make restart` | Запускает `restart.sh` — перезапускает сервер в фоне |

---

## Shell-скрипты

| Скрипт | Что делает |
|---|---|
| `restart.sh` | Убивает старый сервер, освобождает порт 8080, запускает новый в фоне. Вывод сервера подавляется (`/dev/null`). |
| `run_test.sh` | Убивает сервер, чистит `data.bin` и все `.key` файлы, поднимает свежий сервер и запускает Python-тест. По умолчанию `test.py`, можно передать другой: `bash run_test.sh test2.py` |

---

## Python-скрипты (`build/`)

Все скрипты запускаются через `bash run_test.sh [скрипт]` из корня проекта (сервер поднимается автоматически, данные очищаются).

| Скрипт | Что делает |
|---|---|
| `test.py` | Автотест: 6 параллельных клиентов, 6 фаз — регистрация, 12 чатов, приглашения, отправка и чтение сообщений. Используется как регрессионный тест. |
| `test2.py` | Симуляция живого диалога: 4 пользователя (alice, bob, charlie, diana) общаются по заранее написанному сценарию из 12 реплик. Наглядно показывает как работает чат. |
| `dump_db.py` | Парсит `data.bin` и выводит содержимое: пользователи, чаты, ключи чатов, сообщения. Удобно для отладки состояния сервера. |
| `diag.py` | Подключается к живому серверу как `tester1` и запрашивает список всех чатов / своих чатов. Диагностика без пересборки. |
| `diag2.py` | Расширенная диагностика: проверяет структуры ключей чатов и корректность записей в базе. |
